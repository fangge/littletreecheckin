## ADDED Requirements

### Requirement: 响应式应用外壳

系统 SHALL 根据视口宽度自动切换布局模式，在移动端保持现有底部导航布局，在桌面端切换为左侧边栏导航布局。

#### Scenario: 移动端布局保持不变
- **WHEN** 用户在宽度小于 1024px 的设备上访问应用
- **THEN** 应用显示底部导航栏，内容区域占满全宽，与现有移动端体验一致

#### Scenario: 桌面端切换为侧边栏布局
- **WHEN** 用户在宽度大于等于 1024px 的设备上访问应用
- **THEN** 应用显示固定左侧边栏导航（宽度 240px），内容区域占据剩余宽度，底部导航栏隐藏

#### Scenario: 侧边栏导航项展示
- **WHEN** 桌面端侧边栏渲染
- **THEN** 每个导航项以图标 + 文字横排形式展示，当前激活项高亮显示，侧边栏顶部显示应用名称

### Requirement: 内容区域响应式宽度

系统 SHALL 根据视口宽度调整各视图内容区域的最大宽度，充分利用大屏空间。

#### Scenario: Dashboard 树木网格扩展
- **WHEN** 用户在平板端（≥ 768px）查看 Dashboard
- **THEN** 树木卡片网格显示 3 列

#### Scenario: Dashboard 桌面端网格
- **WHEN** 用户在桌面端（≥ 1024px）查看 Dashboard
- **THEN** 树木卡片网格显示 4 列，内容区域最大宽度为 896px（`max-w-4xl`）

#### Scenario: 表单视图内容居中
- **WHEN** 用户在桌面端（≥ 1024px）访问目标设置、打卡、商店等表单视图
- **THEN** 表单内容居中显示，最大宽度为 512px–672px，两侧留有空白，避免内容过宽影响可读性

### Requirement: 浮动元素桌面端适配

系统 SHALL 在桌面端隐藏或调整仅为移动端设计的浮动元素。

#### Scenario: FAB 按钮桌面端隐藏
- **WHEN** 用户在桌面端（≥ 1024px）查看 Dashboard
- **THEN** 右下角浮动添加按钮（FAB）不显示，用户通过侧边栏或页面内按钮添加目标

#### Scenario: 固定底部按钮区域桌面端适配
- **WHEN** 用户在桌面端（≥ 1024px）访问含固定底部按钮的视图（如目标设置）
- **THEN** 底部按钮区域随内容流动（`sticky` 或 `relative`），不遮挡内容，且与内容区域宽度对齐
