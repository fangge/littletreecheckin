## ADDED Requirements

### Requirement: 勋章定义与初始化

系统 SHALL 在数据库中预置所有勋章定义，包含解锁条件配置。

#### Scenario: 勋章数据初始化
- **WHEN** 系统首次部署时执行数据库迁移
- **THEN** 系统预置 9 枚勋章记录，包含名称、图标、颜色、解锁条件（JSON 格式）

#### Scenario: 勋章解锁条件类型
- **WHEN** 系统检查勋章解锁条件
- **THEN** 支持以下条件类型：连续打卡天数、累计完成任务数、累计完成树木数、累计获得果实数

### Requirement: 孩子勋章查询

系统 SHALL 提供孩子勋章列表查询接口，区分已解锁和未解锁状态。

#### Scenario: 获取勋章列表
- **WHEN** 请求 GET /api/v1/children/:childId/medals
- **THEN** 系统返回所有勋章列表，已解锁的勋章包含解锁时间，未解锁的勋章显示解锁条件描述

### Requirement: 勋章自动解锁

系统 SHALL 在任务审核通过后自动检查并解锁符合条件的勋章，无需手动触发。


#### Scenario: 累计任务勋章解锁
- **WHEN** 孩子累计完成任务数达到勋章要求的阈值
- **THEN** 系统自动解锁对应勋章

#### Scenario: 树木完成勋章解锁
- **WHEN** 孩子完成第一棵树木的培育（进度达到 100%）
- **THEN** 系统自动解锁"水果采摘员"勋章

#### Scenario: 重复解锁保护
- **WHEN** 系统检测到某勋章已被解锁
- **THEN** 系统跳过该勋章，不重复创建解锁记录
